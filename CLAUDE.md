## Fase completada: Fase 2 — Configuración del proyecto

### Lo que está instalado y configurado en `client/`:
- Vite 8 + React 19 + TypeScript 6 (scaffold limpio)
- TailwindCSS v4 (via `@tailwindcss/vite`)
- Alias `@/` → `./src/` (en vite.config.ts y tsconfig.app.json)
- ESLint 10 con flat config + eslint-config-prettier
- Prettier (`.prettierrc`)
- Design system base en `src/index.css` (variables CSS: superficie oscura, acento púrpura)

### Dependencias de producción instaladas:
`react-router-dom`, `zustand`, `@tanstack/react-query`, `@tanstack/react-query-devtools`,
`framer-motion`, `axios`, `socket.io-client`, `lucide-react`, `clsx`, `tailwind-merge`,
`date-fns`, `react-hook-form`, `zod`, `@hookform/resolvers`

### Archivos clave creados:
- `src/lib/utils.ts` — función `cn()` para combinar clases Tailwind
- Estructura feature-based en `src/features/` (auth, feed, profile, bands, chat)

### Siguiente fase: Fase 3 — Sistema de diseño visual y primeros componentes UI

## Comandos del proyecto
Todos desde `client/`:
```
npm run dev          # Servidor de desarrollo
npm run build        # Build de producción
npm run lint         # ESLint
npm run lint:fix     # ESLint con autofix
npm run format       # Prettier sobre src/
```

## El usuario trabaja en Windows con PowerShell
Usar `ni <archivo>` en lugar de `touch`, y `New-Item -ItemType Directory` en lugar de `mkdir -p`.
